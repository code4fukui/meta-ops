# meta-ops

1200以上の[Code for FUKUI](https://github.com/code4fukui)リポジトリにおける、大規模なREADMEのメンテナンスと国際化のための運用ツールキット。

このリポジトリには、高品質なバイリンガルのREADMEを生成、監査、デプロイするためのスクリプトと設定が含まれており、多様なオープンソースエコシステム全体で一貫性と明確さを確保します。

## コアワークフロー
スクリプトは、安全な大規模更新のために以下の順序で実行するように設計されています:
1.  **生成:** 生成スクリプトのいずれかを使用して、READMEを作成または更新します。
2.  **監査:** 品質監査スクリプトを実行してレポートを生成し、潜在的な問題を特定します。
3.  **コミット:** `commit_readmes_local.py`を使用して、READMEのみのローカルコミットを作成します。
4.  **レビュー:** ローカルコミットのサンプルを手動で検査し、品質と正確性を確認します。
5.  **プッシュ:** レビュー済みの変更を、制御された安全なバッチでoriginにプッシュします。

## スクリプト (`src/ops/`)
| スクリプト | 説明 |
| --- | --- |
| `generate_readmes_from_codebase.py` | コード構造、依存関係 (`package.json`)、および重要なファイルを分析して、実用的な英語のREADMEを生成します。 |
| `generate_readmes_full_refresh.py` | AWS Bedrock (Claude 3 Haiku) を使用して英語と日本語のREADMEの完全な再生成を行い、スクリーンショットのビジョンサポートも提供します。 |
| `readme_quality_audit.py` | 決定論的なチェックを実行し、AIを使用して生成されたREADMEの品質レポートを作成します。 |

| `push_readmes.py` | レビュー済みのローカルにコミットされたREADMEの変更を、制御された再開可能なバッチでoriginにプッシュします。 |
| `delete_non_main_branches_all.py` | すべてのリポジトリから非デフォルトブランチをクリーンアップするユーティリティスクリプトです。 |
| `regen_all_bulk.py` | プッシュを行わない一括再生成および翻訳ワークフローのためのリファレンススクリプトです。 |

## リポジトリ構造
- `safe_first_candidates.txt`: 初期の低リスクな更新を対象とする、保守的なリポジトリのリスト。
- `src/ops/`: すべてのPython運用スクリプト。
- `CLAUDE.md`: AI支援によるコンテンツ生成のための品質と安全性のガードレール。
- `REPOSITORY_GUIDE.md`: 海外の開発者に向けたCode for FUKUIエコシステムのマップ。

## クイックスタート
```bash

cd ~/meta-ops

# スクリプトのオプションを表示
python3 src/ops/generate_readmes_from_codebase.py --help
```

## 必要条件
- Linuxホスト
- Python 3.11+
- Git
- `~/code4fukui` 配下にチェックアウトされたターゲットリポジトリ
- **オプション:** AWS Bedrockやビジョンサポートを伴うワークフロー用の `boto3`、`ffmpeg`、`Pillow`。

## 安全性のガードレール
- ✅ **直接のプッシュ禁止:** 明示的でレビュー済みのワークフローなしに、デフォルトブランチへプッシュしないでください。
- ✅ **コンテンツの保持:** 既存の重要なプロジェクト説明や帰属（アトリビューション）の行を保持してください。
- ✅ **一貫したライセンス:** ライセンスセクションが直接 `LICENSE` ファイルにリンクしていることを確認してください。
- 🚫 **強制プッシュの禁止:** スクリプトは、強制プッシュや履歴の上書きを避けるように設計されています。
